# Support

Need help with aitap? This file explains the fastest path depending on what you need.

## I Found A Bug

Open a GitHub issue using the bug report template:

- Include exact steps to reproduce
- Mention your OS and Go version
- Mention the provider involved
- Paste logs or sample payloads if they can be shared safely

Issue templates: https://github.com/aniketljoshi/aitap/issues/new/choose

## I Want To Request A Feature

Open a feature request issue when you have:

- A concrete use case
- A gap in the current workflow
- An idea for how the behavior should work

The strongest requests explain the problem first and the implementation idea second.

## I Need Setup Help

Open a help issue if:

- The proxy is not intercepting traffic
- A provider base URL is unclear
- You are not sure which mode to use
- You are contributing and got stuck locally

Please include what you already tried so the next reply can move you forward.

## I Found A Security Problem

Do not open a public issue.

Follow the instructions in [SECURITY.md](SECURITY.md) and report it privately to
[aniket@aitap.dev](mailto:aniket@aitap.dev).

## Before Opening Anything

Please check:

- [README.md](README.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- Existing GitHub issues

That keeps reports focused and makes it easier to respond quickly.
